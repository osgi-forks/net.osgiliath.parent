Purpose

The route module allow you to transform and route your messages see and read the Enterprise Integration Pattern In order to use it, generate the skeleton with the Osgiliath route archetype, then, create a simple route: {% codeblock lang:java %} public class HelloRoute extends RouteBuilder { private DataFormat helloObjectJSonFormat = new JacksonDataFormat( HelloObject.class); //sample transformation of Json object JAXBContext ctx = JAXBContext .newInstance(new Class[] { HelloObject.class }); DataFormat jaxBDataFormat = new JaxbDataFormat(ctx);//Xml transformation of an object from("{{hello.helloJaxRSEntryPoint}}") .filter(header("webSocketMsgType").isNotEqualTo("heartBeat")) .unmarshal(helloObjectJSonFormat) .marshal(jaxBDataFormat) .to("{{net.osgiliath.hello.business.url.helloservice}}/hello") } {% endcodeblock %} The URL/entrypoints in bracket {{}} are from the cfg file of the feature module while executing on runtime and from the src/test/resources/cfg one when unit testing. Then, you've to reference this route on the camelContext section of the blueprint.xml file and in the src/test/resources/spring (for testing). {% codeblock lang:xml %} {% endcodeblock %} You can also use @ContextName camel-cdi annotation and reference component in a configuration class For more informations, check the hello example on github
